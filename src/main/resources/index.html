CrudX Starter is a lightweight, high-performance Spring Boot framework built to simplify CRUD operations across multiple databases. It eliminates repetitive configurations and enables rapid development using a clean, modular architecture.

CrudX provides auto-configuration for both SQL and NoSQL databases, audit tracking, and a ready-to-use repository layer ‚Äî all while keeping your codebase minimal and maintainable. Start building scalable CRUD systems faster and smarter.

‚ö° Quick Start (5 Steps)

Step 1: Add Dependencies

Add the core dependency and your database driver: 

implementation 'io.github.sachinnimbal:crudx-starter:loading...'

Step 2: Enable CrudX

Annotate your main Spring Boot class:

@SpringBootApplication
@CrudX
public class MyApp { ... }

Step 3: Configure Database

Add application.yml for MySQL, PostgreSQL, or MongoDB: 

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/company_db
    username: root
    password: root123

Step 4: Create Entity

Extend the CrudX base model for your database type: 

public class Employee extends CrudXMySQLEntity<Long> {
  private String name;
  private String email;
}

Step 5: Create Controller

Extend CrudXController to automatically enable CRUD endpoints: 

@RestController
@RequestMapping("/api/employees")
public class EmployeeController extends CrudXController<Employee, Long> { }
About Memory Metrics: Memory values show thread-allocated memory during request processing. First requests may show higher values due to JVM warm-up.

CrudX automatically generates CRUD APIs, repositories, and services for all annotated entities ‚Äî no manual boilerplate required!
